# 프로젝트 3단계 승인 운영 점검 체크리스트

목적: 프로젝트 승인(1차/2차/3차) 기능이 운영에 안전하게 반영되었는지 검증한다.

## 0) 사전 준비

- [ ] 최신 코드 반영 후 브라우저 강력 새로고침(캐시 비우기) 완료
- [ ] 대상 환경에서 관리자 계정 로그인 가능
- [ ] 아래 역할 계정 준비
  - [ ] Staff(신청자)
  - [ ] Manager(1차 승인자)
  - [ ] Director(2차 승인자)
  - [ ] Top Manager(3차 최종 승인자)

## 1) DB 스키마 적용 확인

- [ ] `docs/sql/dev_add_registered_projects_approval_3step.sql` 실행 완료
- [ ] `registered_projects` 컬럼 존재 확인
  - [ ] `reg_pa3_id`
  - [ ] `reg_pa3_name`
  - [ ] `second_approved_at`
  - [ ] `second_approved_by`
  - [ ] `second_approved_by_name`
- [ ] 기존 SQL도 적용 상태 확인
  - [ ] `dev_add_registered_projects_conditional_approval.sql`
  - [ ] `dev_add_registered_projects_order_route_fields.sql`
  - [ ] `dev_setup_project_contract_storage.sql`

## 2) 기본 3단계 승인 플로우 점검 (정상 케이스)

### 2-1. Staff 승인요청
- [ ] Staff가 프로젝트 등록 후 승인요청
- [ ] 상태가 `승인대기`로 표시
- [ ] 단계가 `1차 승인 대기`로 표시
- [ ] Manager에게 승인요청 알림 도착

### 2-2. Manager 1차 승인
- [ ] Manager 승인 목록에서 해당 프로젝트 확인 가능
- [ ] 상세에서 승인 처리 가능
- [ ] 처리 후 단계가 `2차 승인 대기`로 전환
- [ ] Director에게 다음 단계 알림 도착

### 2-3. Director 2차 승인
- [ ] Director 승인 목록에서 해당 프로젝트 확인 가능
- [ ] 상세에서 승인 처리 가능
- [ ] 처리 후 단계가 `3차 승인 대기`로 전환
- [ ] Top Manager에게 다음 단계 알림 도착

### 2-4. Top Manager 3차 최종 승인
- [ ] Top Manager 승인 목록에서 해당 프로젝트 확인 가능
- [ ] 상세에서 승인 처리 가능
- [ ] 상태가 `승인완료`로 전환
- [ ] 신청자(Staff)에게 최종 승인 알림 도착

## 3) 반려 플로우 점검

- [ ] Manager/Director/Top Manager 각 단계에서 반려 가능
- [ ] 반려 시 사유 입력 필수 동작 확인
- [ ] 상태가 `반려`로 전환
- [ ] 신청자에게 반려 알림 도착
- [ ] 알림 문구에 재제출 안내 포함 확인

## 4) 조건부 승인(계약서 미첨부) 점검

- [ ] 계약서 없이 승인요청 시 합의근거 파일 + 사유 미입력이면 차단
- [ ] 합의근거 파일 + 사유 입력 시 승인요청 가능
- [ ] 최종 승인 시 상태가 `조건부승인`으로 표시
- [ ] 이후 계약서 업로드 시 조건부 해소 저장 로직 동작 확인

## 5) 배지/카운트 점검

- [ ] 사이드바 `Approval` 배지 숫자 정상
  - [ ] 타임시트 + 프로젝트 합산값
  - [ ] 프로젝트는 현재 사용자에게 승인 권한 있는 건만 집계
- [ ] Approval 탭 카운트 정상
  - [ ] `타임시트 승인 (n)`
  - [ ] `프로젝트 승인 (m)`
- [ ] Manager/Director/Top Manager 각 계정에서 숫자 차등 반영 확인

## 6) 권한/범위 점검

- [ ] Staff: 본인 건만 조회/수정/승인요청 가능
- [ ] Manager/Director/Top Manager: 소속 범위(본인+소속)만 노출
- [ ] Admin: 전체 조회 가능
- [ ] 계약서 관리현황 연결해제 권한 확인
  - [ ] 등록자/최종승인자/Admin 허용
  - [ ] 그 외 사용자 차단

## 7) 첨부파일 점검

- [ ] 계약서/합의근거/수주경로 증빙 업로드 가능
- [ ] 첨부파일 `열기` 동작
- [ ] 파일 교체 시 이전 파일 정리(스토리지 삭제 시도) 확인
- [ ] 계약서 관리현황 모달에서 검색/열기/연결해제 동작

## 8) 회귀 점검 (타임시트 운영 무영향)

- [ ] 타임시트 승인 플로우(1차/2차) 기존 동작 유지
- [ ] 타임시트 알림 문구/배지 계산 이상 없음
- [ ] 타임시트 화면에서 프로젝트 3단계 변경으로 인한 UI/권한 영향 없음

## 9) 실패 시 확인 포인트

- [ ] SQL 누락 여부(특히 `reg_pa3_*`, `second_approved_*`)
- [ ] 브라우저 캐시 미반영 여부
- [ ] 사용자 마스터 정보(역할, 부서/본부 매핑) 누락 여부
- [ ] 알림 테이블 쓰기 권한/정책 확인
- [ ] 스토리지 버킷 정책 및 URL 접근 권한 확인

## 10) 최종 배포 승인 기준

- [ ] 3단계 승인 정상
- [ ] 반려/재제출 정상
- [ ] 조건부 승인 및 사후 계약서 관리 정상
- [ ] 배지/알림/권한/범위 정상
- [ ] 타임시트 회귀 테스트 통과

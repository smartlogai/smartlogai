# 일괄기록 입력/수정/승인요청 사용자 매뉴얼

## 1) 문서 목적

본 문서는 신규 기능인 **일괄기록**을 현업에서 빠르게 사용하도록 돕기 위한 교육용 자료입니다.  
아래 3가지를 중심으로 설명합니다.

- 일괄기록 입력 방법
- 상세보기 후 수정 방법
- 승인요청(제출) 방법과 주의사항

---

## 2) 대상 사용자

- 컨설턴트(업무 기록 작성자)
- 팀장/승인자(검토자)
- 운영/교육 담당자

---

## 3) 기능 개요 (핵심 이해)

일괄기록은 하루 업무를 **행 단위**로 여러 건 입력한 뒤, 하나의 타임시트로 저장/제출하는 방식입니다.

- 각 행에는 `대분류/소분류/고객사/업무기록/시간`이 들어갑니다.
- 저장 시 내부적으로 `헤더 1건 + 상세행 N건` 구조로 보관됩니다.
- 목록에서는 일괄기록이 1건으로 보이고, 상세보기에서 행 단위 내역을 확인할 수 있습니다.

---

## 4) 화면 구성 이해

`Time Entry` 화면에서 입력 방식을 선택합니다.

- `개별기록`: 기존 방식(건별 기록)
- `일괄기록`: 신규 방식(행 단위 다건 입력)

일괄기록 모드 주요 영역:

- 좌/상단: 시간 시각화(타임라인)
- 우/하단: 행 입력 카드(대분류, 소분류, 고객사, 업무기록, 시간)
- 하단 버튼: 임시저장, 제출

---

## 5) 일괄기록 입력 요령 (실무 권장 순서)

### 5-1. 기본 입력 순서

행 1개 기준 권장 순서:

1. `대분류` 선택
2. 필요 시 `소분류` 선택
3. 고객업무면 `고객사` 선택 또는 직접입력
4. `업무기록` 입력
5. `시작시간` 입력
6. `소요시간` 선택(10분 단위 권장)
7. `확정` 클릭

### 5-2. 시간 입력 원칙

- 시작시간은 `HH:mm` 형식으로 입력
- 소요시간은 10분 단위로 조정
- 다음 행 추가 시 시작시간은 이전 행 종료시간을 기준으로 잡는 것을 권장

### 5-3. 고객사 입력 원칙

- 회사내부업무: 고객사 비활성
- 일반통관/프로젝트/기타 고객업무: 고객사 사용
- 신규 업체 방문 등 DB 미등록 업체: 고객사 직접입력 사용 가능

### 5-4. 프로젝트업무 입력 원칙

- 대분류를 프로젝트업무로 선택
- 소분류 대신 프로젝트코드 중심으로 입력/확인

---

## 6) 임시저장과 복구

일괄기록은 작성 중 이탈에 대비해 임시저장을 사용합니다.

- `임시저장`: 작성 중 상태 저장
- `복구`: 최근 임시저장 상태를 입력 화면에 다시 로드

권장:

- 장시간 작성 시 중간중간 임시저장
- 브라우저 새로고침/재접속 후 `복구` 버튼으로 이어쓰기

---

## 7) 상세보기 후 수정 방법

경로: `My Time Sheet` -> `일괄기록` -> 대상 건 `상세보기` -> `수정`

수정 시 확인 포인트:

- 모든 행이 로드되었는지 먼저 확인
- 행별 대분류/소분류/고객사/업무기록/시간을 수정
- 불필요 행 삭제, 누락 행 추가
- 최종 합계 시간과 행별 시간이 의도대로 맞는지 점검

권장 검토 체크:

- 행 누락 여부
- 시간 중복/역전 여부
- 고객사/프로젝트코드 오입력 여부

---

## 8) 승인요청(제출) 방법

작성 완료 후 제출 절차:

1. 필수값 누락 여부 확인
2. `제출` 클릭
3. 상태가 `submitted`(검토 대기)로 변경되었는지 확인

승인 흐름:

- 일괄기록: 1차 승인으로 종결
- 개별기록(일반자문/프로젝트업무 일부): 조직 규칙에 따라 추가 승인 흐름 존재

---

## 9) 승인 화면에서 확인 방법

`Approval` 화면에서 탭별로 확인합니다.

- `개별기록 승인`
- `일괄기록 승인`

목록에서 `상세보기`를 눌러 행 단위 내역을 검토합니다.  
필요 시 컬럼 헤더 클릭으로 정렬(오름/내림) 가능합니다.

---

## 10) 자주 발생하는 실수와 예방

### 실수 1: 시간만 입력하고 `확정`을 누르지 않음
- 예방: 각 행 입력 후 반드시 `확정` 클릭

### 실수 2: 고객업무인데 고객사 누락
- 예방: 대분류 선택 직후 고객사 입력 상태 확인

### 실수 3: 임시저장 후 복구 없이 새로 작성
- 예방: 재진입 시 `복구` 버튼 먼저 확인

### 실수 4: 제출 전 행 누락
- 예방: 상세행 건수와 업무기록 건수 대조 후 제출

---

## 11) 교육 운영용 체크리스트

교육 담당자는 아래 항목으로 실습 완료 여부를 확인합니다.

- [ ] 일괄기록 모드 진입 가능
- [ ] 3행 이상 입력 후 확정 가능
- [ ] 임시저장/복구 동작 확인
- [ ] 상세보기 -> 수정 -> 재저장 가능
- [ ] 제출 후 승인대기 상태 전환 확인
- [ ] 승인 화면에서 상세행 검토 가능

---

## 12) 교육용 10분 실습 시나리오

1. `개별기록`/`일괄기록` 차이 설명 (1분)
2. 일괄기록 3행 입력 (3분)
3. 임시저장 후 복구 (2분)
4. 상세보기 -> 수정 1건 변경 (2분)
5. 제출 및 승인 화면 확인 (2분)

---

## 13) 문의/장애 보고 시 전달 정보

이슈 보고 시 아래 정보 포함:

- 사용자명/권한(작성자, 승인자)
- 발생 메뉴(`Time Entry`, `My Time Sheet`, `Approval`)
- 수행 동작(예: 상세보기 -> 수정 클릭)
- 기대 결과 vs 실제 결과
- 발생 시각, 스크린샷, 콘솔 오류 메시지

## 부록 A) 입력 품질 가이드 (권장)

- 업무기록은 결과 중심으로 1~2문장 작성
- 프로젝트업무는 프로젝트코드 확인 후 저장
- 하루 종료 전 제출 전검토(행수/시간합/고객사) 수행

---

## 부록 B) 용어 정리

- **개별기록**: 기존 건별 기록 방식
- **일괄기록**: 여러 업무를 행 단위로 한 번에 작성하는 방식
- **임시저장**: 제출 전 중간 저장
- **제출**: 승인요청 상태로 전환
